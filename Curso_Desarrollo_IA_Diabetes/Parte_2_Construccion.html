Machine Learning

Capítulo 5: Entrenamiento del "Cerebro" Artificial

5.1 Train/Test Split

Aprenderemos la regla de oro del Data Science: nunca evaluar con los datos de práctica. Dividiremos nuestro dataset 80/20 para garantizar una evaluación honesta del rendimiento.

5.2 Serialización con Joblib

Una vez que el modelo "aprende", congelaremos su conocimiento en un archivo .pkl. Esto permite que la IA funcione sin re-entrenarse cada vez que encendemos el servidor.

API Design

Capítulo 6: Integración y Endpoints Inteligentes

6.1 Validación Estricta con Pydantic

En medicina, los datos erróneos son fatales. Crearemos esquemas DTO (Data Transfer Objects) que rechacen automáticamente valores imposibles (ej. edad negativa) antes de que toquen nuestra IA.

6.2 Ruta POST de Predicción

Desarrollaremos el endpoint /predict que orquesta todo: recibe datos, los convierte a matrices NumPy, consulta al modelo y devuelve el pronóstico clínico en JSON.

Frontend Framework

Capítulo 7: Frontend Reactivo con Vite

7.1 Velocidad de Desarrollo

Abandonaremos herramientas lentas. Usaremos Vite para un desarrollo instantáneo. Configuraremos TypeScript para añadir seguridad de tipos a nuestros formularios médicos.

7.2 Arquitectura de Componentes

Estructuraremos nuestra app en componentes modulares, preparándonos para una escalabilidad masiva.

UI / UX

Capítulo 8: Psicología del Diseño Médico (Glassmorphism)

8.1 Estética de Confianza

Diseñaremos una interfaz que elimine la ansiedad. Usaremos paletas de colores oscuros, efectos de cristal esmerilado (Glassmorphism) y tipografía geométrica (Outfit) para transmitir modernidad y precisión.

8.2 Micro-interacciones con CSS

Aprenderemos a usar gradientes y animaciones sutiles que guíen al usuario a través del complejo formulario de datos clínicos sin abrumarlo.
